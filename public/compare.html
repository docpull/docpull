docpull vs Alternatives

docpull is purpose-built for AI agents. Unlike traditional PDF extraction tools, it requires no accounts, no API keys, and no subscriptions. Any agent with a Base wallet and USDC can call it autonomously via the x402 v2 payment protocol.

Why agents prefer docpull

Traditional PDF APIs require a human developer to sign up, manage API keys, and configure billing. This creates friction that prevents AI agents from autonomously accessing PDF extraction capabilities.

docpull eliminates this entirely. An agent discovers docpull via the CDP Bazaar or MCP registry, calls /probe to check cost, then calls /extract — the x402 protocol handles payment settlement on Base mainnet with no human intervention required.

Unlike structured JSON extraction tools (which require custom extraction instructions), docpull returns clean Markdown that preserves document structure and is immediately usable in agent pipelines for RAG, summarization, or further analysis.

When to use docpull

  • Your agent needs to ingest PDF documents autonomously
  • You want pay-per-use pricing without subscription overhead
  • You are building an x402-native agent pipeline on Base
  • You need structured Markdown output (not raw text)
  • You want MCP tool integration for Claude, ChatGPT, or other MCP clients

When to use alternatives

  • You need structured JSON extraction with custom field mapping (consider Strale)
  • You are processing PDFs in a traditional server-side pipeline (consider pdfplumber)
  • You need OCR for scanned documents (docpull uses pdfjs-dist, which requires selectable text)
