docpull converts any publicly accessible PDF URL into clean structured Markdown. Pay $0.001 USDC per page via x402 v2 on Base mainnet — no accounts, no API keys, no subscriptions.

Quickstart

1. Check cost (free)

curl "https://docpull.ai/probe?url=https://example.com/doc.pdf"
// { "pageCount": 5, "costUSDC": "0.005000", "pricePerPage": "0.001 USDC" }

2. Install x402 client

npm install @x402/fetch @x402/evm viem

3. Extract with automatic payment

import { x402Client, wrapFetchWithPayment } from "@x402/fetch";
import { ExactEvmScheme } from "@x402/evm/exact/client";
import { privateKeyToAccount } from "viem/accounts";

const signer = privateKeyToAccount(process.env.EVM_PRIVATE_KEY);
const client = new x402Client().register("eip155:*", new ExactEvmScheme(signer));
const fetchWithPayment = wrapFetchWithPayment(fetch, client);

const res = await fetchWithPayment("https://docpull.ai/extract", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ url: "https://example.com/doc.pdf" }),
});
const { markdown, pageCount } = await res.json();

Authentication

docpull uses the x402 v2 protocol instead of API keys. The /health, /probe, and /status endpoints are always free. The /extract endpoint requires a USDC payment on Base mainnet.

No account creation, no API key management, no subscriptions. Any agent with a funded Base wallet can call docpull immediately.

x402 payment flow

When you call POST /extract without a payment header:

  1. Server returns 402 Payment Required
  2. Response includes PAYMENT-REQUIRED header (base64 JSON with payment instructions)
  3. x402 client decodes requirements and signs USDC transfer on Base mainnet
  4. CDP facilitator verifies and settles the payment
  5. Client retries with X-PAYMENT header
  6. Server returns the extracted Markdown

Network: Base mainnet (eip155:8453) · Asset: USDC (0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913)

MCP integration

docpull exposes an MCP server at https://docpull.ai/mcp via Streamable HTTP transport.

{
  "type": "streamable-http",
  "url": "https://docpull.ai/mcp"
}

Available tools: probe_pdf, extract_pdf, health_check

Error handling

All errors return structured JSON with error and code fields:

{ "error": "url body field required", "code": "MISSING_URL" }

Error codes: MISSING_URL, PDF_FETCH_FAILED, NOT_FOUND, INTERNAL_ERROR, RATE_LIMIT_EXCEEDED

On 402: decode the PAYMENT-REQUIRED header and send payment via an x402-compatible client.

On 429: check X-RateLimit-Reset header for retry timestamp. Use exponential backoff starting at 1 second.

Rate limits

1000 requests per hour. Rate limit headers are included on every response:

  • X-RateLimit-Limit — requests allowed per hour
  • X-RateLimit-Remaining — requests remaining in current window
  • X-RateLimit-Reset — Unix timestamp when limit resets
